<h1 align="center">Учебный центр</h1>

<p align="center">
  Ниже представлена концептуальная модель пользовательского интерфейса
</p>

Двигатель програсса это пользователь/заказчик, а значит сценарии использования стоит разделить по потребностям разных ролей.

## Роли пользователей
* Администратор учебного центра
* Преподаватель
* Ученик

## Администратор учебного центра
### Управление учениками 
* Добавление ученика
* Удаление ученика
* Редактирование данных ученика
* Запись ученика на курс
### Управление преподавателями
* Добавление преподавателя
* Удаление преподавателя
* Редактирование данных преподавателя
### Управление курсами
* Добавление курса
* Удаление курса
* Редактирование курса
### Формирование расписания
* Составление расписания занятий для курса
* Просмотр расписания по интервалу времени
### Получение списков
* Получение списка учеников по курсам
* Получение истории обучения
* Получение списка преподавателей, включая распределение по курсам

## Преподаватель
### Просмотр расписания
* Получение расписания на заданный интервал времени
### Просмотр списков студентов
* Получение списка обучающихся по курсу

## Ученик
### Просмотр личного расписания
* Получение расписания на заданный интервал времени
### Просмотр истории обучения
* Просмотр истории прохождения курсов
