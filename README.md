<p align="center">
  Ниже представлена концептуальная модель пользовательского интерфейса
</p>

Двигатель програсса это пользователь/заказчик, а значит сценарии использования стоит разделить по потребностям разных ролей.

## Роли пользователей
- [Администратор учебного центра](#администратор-учебного-центра)
- [Преподаватель](#преподаватель)
- [Ученик](#ученик)

## Администратор учебного центра
### Управление учениками 
* Добавление ученика
* Удаление ученика
* Редактирование данных ученика
* Запись ученика на курс
* Снятие ученика с курса
* Уведомления ученику
### Управление преподавателями
* Добавление преподавателя
* Удаление преподавателя
* Редактирование данных преподавателя
* Назначение преподавателя на курс
### Управление курсами
* Добавление курса
* Удаление/архивация курса 
* Редактирование курса
* Уведомление участникам курса
### Формирование расписания
* Составление расписания занятий для курса
* Перенос занятия, отмена занятия
* Просмотр расписания по интервалу времени
### Получение списков
* Получение списка учеников по курсам
* Получение истории обучения / просмотр ЛК ученика
* Получение списка преподавателей, включая распределение по курсам

## Преподаватель
### Личные данные
* Просмотр/редактирование профиля
### Просмотр расписания
* Получение расписания на заданный интервал времени
* Просмотр деталей занятия
* Подтверждение/отметка проведения занятия
* Запрос замены/переноса занятия
* Добавление комментариев/заметок по занятию (дз например)
### Взаимодействие с учениками
* Получение списка обучающихся по курсу
* Просмотр карточки ученика в рамках своего курса
* Отметка посещаемости

## Ученик
### Личные данные
* Просмотр/редактирование профиля
### Просмотр личного расписания
* Получение расписания на заданный интервал времени
* Просмотр карточки курса
* Просмотр деталей занятия
### Просмотр истории обучения
* Просмотр истории прохождения курсов
### Запись на обучение
* Подача заявки на курс
